# Website Design Principles

This document dictates the design guidelines and aesthetics for the OKLAD (Oklahoma Labeled AI Dataset) website. The goal is to provide a premium, unforgettable user experience that immediately impresses researchers and data scientists.

## 1. Rich Aesthetics
The website must look "wow" at first glance. We avoid generic bootstrap colors (plain red, blue, green) and instead use a curated, harmonious custom palette. A sleek dark mode or a very high-contrast modern light theme with deep, rich accents is required. 

## 2. Glassmorphism & Depth
Use glassmorphism (translucent backgrounds with background blur) for overlay elements, navigation bars, and cards to create a sense of depth and modernity. Add subtle drop shadows to create floating effects.

## 3. Modern Typography
Move away from default fonts. Use sleek, modern Google Fonts like **Inter**, **Outfit**, or **Roboto**. Ensure high readability for dataset statistics, and impactful weights (bold/black) for headings.

## 4. Dynamic Micro-animations
The interface should feel alive. Implement:
- Smooth hover states on all interactive elements (buttons, cards, links).
- Fade-in animations as the user scrolls down the page.
- Animated loading states or visual transitions when navigating.

## 5. Engaging Data Visualization
Since this is an AI / Earthquake dataset website:
- Use interactive libraries (Chart.js / Plotly / D3) where possible instead of static images.
- Visualizations should follow the overarching color palette. 

## 6. SEO & Accessibility Best Practices
- Every page must feature proper Semantic HTML (`<header>`, `<main>`, `<footer>`, `<section>`).
- Alt text for all images.
- Meta descriptions and Title tags.
- Unique IDs for interactive elements.
